# Rules of Engagement

## Mandatory
- Only test assets explicitly listed as in-scope.
- Respect testing windows, rate limits, and forbidden vectors.
- Stop immediately when production stability is at risk.
- Never access, modify, or retain unnecessary user data.

## Operational Guardrails
- Every scan must be tied to a program and scope record.
- Every finding must include reproducible evidence.
- Keep immutable logs of tool output and timestamps.
- Keep proof-of-concept minimally invasive.

## Report Quality Standard
- Clear title with vulnerability class and affected component.
- Repro steps must work from a clean environment.
- Demonstrate business impact with concrete outcome.
- Include remediation guidance that is actionable.
